# /write_pr_summary

## 목표

현재 브랜치의 모든 커밋과 변경사항을 분석하여 명확하고 구조화된 PR 요약을 생성합니다.

## 실행 단계

### 1단계: 브랜치 분석

```bash
# 현재 브랜치와 기본 브랜치 확인
git branch --show-current
git log --oneline main..HEAD

# 변경된 파일 확인
git diff main...HEAD --stat

# 상세 변경사항 확인
git diff main...HEAD
```

### 2단계: 변경사항 분류

변경 유형을 식별합니다:
- **새 기능**: 새로운 에셋, 컴포넌트, 통합
- **버그 수정**: 기존 기능의 오류 수정
- **리팩토링**: 기능 변경 없는 코드 개선
- **설정 변경**: YAML, 환경 변수, pyproject.toml 수정
- **테스트**: 테스트 추가 또는 수정
- **문서**: 문서 업데이트

### 3단계: PR 요약 생성

다음 형식으로 PR 요약을 생성합니다:

```md
## 요약

[1-3줄의 변경사항 요약. 무엇을 왜 변경했는지 설명]

## 주요 변경사항

- [변경 1: 구체적 파일/컴포넌트와 변경 내용]
- [변경 2: ...]

## 테스트 방법

- [ ] `just ruff` 통과
- [ ] `just pyright` 통과
- [ ] 관련 테스트 통과
- [ ] [추가 검증 항목...]
```

### 4단계: 출력

생성된 요약을 사용자에게 표시합니다. 사용자가 승인하면 `gh pr edit`으로 PR에 적용할 수 있습니다.

## 주의사항

- 최신 커밋만이 아닌, 브랜치의 **모든** 커밋을 분석합니다.
- 변경의 "무엇"보다 "왜"에 초점을 맞춥니다.
- DOP 프로젝트의 맥락에서 비즈니스 가치를 설명합니다.
