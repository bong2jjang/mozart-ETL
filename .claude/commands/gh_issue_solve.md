# /gh_issue_solve

## 목표

GitHub 이슈를 분석하여 DOP 프로젝트 맥락에서 구현 계획을 수립합니다.

## 사용법

```
/gh_issue_solve 123
/gh_issue_solve https://github.com/owner/repo/issues/123
```

## 실행 단계

### 1단계: 이슈 정보 가져오기

```bash
gh issue view <번호> --json title,body,labels,assignees,comments
```

이슈 제목, 본문, 레이블, 댓글을 분석합니다.

### 2단계: 코드베이스 조사

이슈와 관련된 코드를 탐색합니다:
- 관련 에셋, 컴포넌트, 라이브러리 코드 검색
- 기존 패턴 및 유사 구현 확인
- 영향을 받는 파일 범위 파악

### 3단계: 구현 계획 수립

EnterPlanMode를 사용하여 구현 계획을 작성합니다:

```md
# 구현 계획: [이슈 제목]

> **이슈:** #<번호>

## 문제 분석
[이슈에서 설명하는 문제/요청 사항 요약]

## 코드베이스 조사 결과
[관련 파일, 패턴, 기존 구현 분석]

## 구현 단계
1. [단계 1: 구체적 파일과 변경 내용]
2. [단계 2: ...]

## 테스트 계획
- [테스트 1]
- [테스트 2]

## 영향 범위
- 변경되는 파일 목록
- 영향 받는 에셋/컴포넌트
```

### 4단계: 사용자 승인

계획을 사용자에게 제시하고 승인을 받은 후 구현을 시작합니다.

## 주의사항

- 이슈 내용만으로 불충분한 경우 사용자에게 추가 정보를 요청합니다.
- 구현 전에 항상 계획을 먼저 수립합니다.
- DOP의 기존 패턴과 일관성을 유지합니다.
